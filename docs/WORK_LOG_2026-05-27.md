# SECTION 2: SECURITY FIXES

Security was the highest priority due to the potential for data breaches and unauthorized access.

## 2.1 Token Storage Vulnerability (CRITICAL - FIXED)

### Problem Description

The application was storing authentication tokens in sessionStorage, which is accessible to JavaScript running on the page. This creates a vulnerability where malicious scripts injected via XSS (Cross-Site Scripting) could steal these tokens and gain unauthorized access to user accounts.

The vulnerable code pattern found in multiple files:

```typescript
// VULNERABLE CODE - DO NOT USE
sessionStorage.setItem('accessToken', token);
const token = sessionStorage.getItem('accessToken');
```

### Files Affected

1. `frontend/src/stores/authStore.ts` - Main auth state storage
2. `frontend/src/services/api.ts` - API request interceptors
3. `frontend/src/services/walletApi.ts` - Wallet service API calls

### Solution Implemented

We implemented a secure storage solution using the secureStorage utility from `@/lib/security`, which provides encrypted storage that is not accessible to JavaScript in the same way as sessionStorage.

#### Changes to authStore.ts

```typescript
// BEFORE (VULNERABLE)
const useAuthStore = create(
  persist(
    (set, get) => ({
      tokens: null,
      setTokens: (tokens) => set({ tokens }),
    }),
    {
      name: 'auth-storage',
      storage: createJSONStorage(() => sessionStorage), // VULNERABLE
    }
  )
);

// AFTER (SECURE)
import { secureStorage } from '@/lib/security';

// Custom secure storage adapter
const secureAuthStorage = {
  getItem: (name: string) => {
    try {
      const value = secureStorage.getItem(name);
      return value ? JSON.parse(value) : null;
    } catch {
      return null;
    }
  },
  setItem: (name: string, value: any) => {
    try {
      secureStorage.setItem(name, JSON.stringify(value));
    } catch (error) {
      console.error('Failed to save auth state:', error);
    }
  },
  removeItem: (name: string) => {
    try {
      secureStorage.removeItem(name);
    } catch (error) {
      console.error('Failed to remove auth state:', error);
    }
  },
};

const useAuthStore = create(
  persist(
    (set, get) => ({
      tokens: null,
      setTokens: (tokens) => set({ tokens }),
    }),
    {
      name: 'auth-storage',
      storage: createJSONStorage(() => secureAuthStorage), // SECURE
    }
  )
);
```

## 2.2 Financial Data in Console Logs (HIGH - FIXED)

### Problem Description

The ProviderEarningsPage component was logging full socket event data to the console, which could include sensitive financial information such as IBAN numbers, bank account details, and transaction amounts. This data should never appear in browser console logs as it could be accessed by malicious browser extensions or debug tools.

### Location
`frontend/src/pages/provider/ProviderEarningsPage.tsx` - Lines 231-285

### Solution

Sanitized all console.log statements and socket event handlers to only log minimal, non-sensitive information:

```typescript
// BEFORE (LEAKS SENSITIVE DATA)
socketService.onBookingStatusChanged((data) => {
  console.log('Booking status changed:', data); // Could contain sensitive info
});

// AFTER (SANITIZED)
socketService.onBookingStatusChanged((data) => {
  console.log('Booking event received:', data.type); // Only logs event type
});
```

Also ensured that any transaction or financial data passed through socket events is never logged, only the event type and non-sensitive metadata.

## 2.3 Missing File Validation on Uploads (HIGH - FIXED)

### Problem Description

The ProviderVerificationPage component allowed file uploads with minimal validation. The only check was `file.size === 0`, which meant:
- No validation of file types (could upload executables)
- No size limits (could upload arbitrarily large files)
- No MIME type verification (could disguise file types)

This creates security vulnerabilities including:
- Malware uploads
- Denial of service via large files
- File type bypass attacks

### Location
`frontend/src/pages/provider/ProviderVerificationPage.tsx` - Lines 200-276

### Solution

Added comprehensive file validation:

```typescript
// Define allowed file types for identity verification
const ALLOWED_MIME_TYPES = [
  'image/jpeg',    // .jpg, .jpeg
  'image/png',     // .png
  'application/pdf' // .pdf
];

// Maximum file size: 10MB (prevents denial of service)
const MAX_FILE_SIZE = 10 * 1024 * 1024; // 10MB in bytes
const MIN_FILE_SIZE = 1024; // Minimum 1KB (prevents empty files)

// Validation function
const validateFile = (file: File): { valid: boolean; error?: string } => {
  // Check if file exists
  if (!file) {
    return { valid: false, error: 'No file selected' };
  }

  // Check file size - minimum
  if (file.size < MIN_FILE_SIZE) {
    return { valid: false, error: 'File is too small. Minimum size is 1KB.' };
  }

  // Check file size - maximum
  if (file.size > MAX_FILE_SIZE) {
    return { valid: false, error: 'File is too large. Maximum size is 10MB.' };
  }

  // Check MIME type
  if (!ALLOWED_MIME_TYPES.includes(file.type)) {
    return { valid: false, error: 'Invalid file type. Please upload PDF, PNG, or JPG files only.' };
  }

  // Additional security: Check file extension matches MIME type
  const extension = file.name.split('.').pop()?.toLowerCase();
  const mimeToExtension: Record<string, string[]> = {
    'image/jpeg': ['jpg', 'jpeg'],
    'image/png': ['png'],
    'application/pdf': ['pdf']
  };

  const allowedExtensions = mimeToExtension[file.type] || [];
  if (extension && !allowedExtensions.includes(extension)) {
    return { valid: false, error: 'File extension does not match file type. Please rename your file.' };
  }

  return { valid: true };
};

// Usage in upload handler
const handleFileUpload = async (file: File, documentType: string) => {
  // Validate first
  const validation = validateFile(file);
  if (!validation.valid) {
    toast?.addToast({
      title: 'Invalid File',
      description: validation.error,
      variant: 'error'
    });
    return;
  }

  // Proceed with upload...
  try {
    const formData = new FormData();
    formData.append('file', file);
    formData.append('documentType', documentType);

    // Create upload with progress tracking
    const response = await uploadDocument(formData);

    toast?.addToast({
      title: 'Upload Successful',
      description: `${documentType} uploaded successfully`,
      variant: 'success'
    });
  } catch (error) {
    toast?.addToast({
      title: 'Upload Failed',
      description: error instanceof Error ? error.message : 'Unknown error',
      variant: 'error'
    });
  }
};
```

### Additional Security Measures

1. **File Content Validation**: The backend should also validate file content matches declared MIME type
2. **Virus Scanning**: Recommend implementing virus scanning on uploaded files
3. **Storage Isolation**: Uploaded files should be stored in a separate, non-executable location
4. **Access Control**: Files should only be accessible to authorized users

# SECTION 3: ADMIN-PROVIDER REAL-TIME COMMUNICATION

## 3.1 Overview

A critical requirement for the provider dashboard was real-time synchronization between admin and provider actions. When an admin approves a provider, the provider should immediately see the update without refreshing the page. Similarly, when a provider submits verification, admins should see the notification in real-time.

We implemented this using Socket.io with custom event handlers.

## 3.2 Socket Events Architecture

### Event Types

We defined two categories of events:
1. **Server-to-Client Events**: Server pushes updates to connected clients
2. **Client-to-Server Events**: Clients send requests to server

```typescript
export const approveProvider = asyncHandler(async (req: Request, res: Response) => {
  const { id } = req.params;
  const { notes } = req.body;
  const adminUser = req.user as any;

  // Find provider
  const provider = await ProviderProfile.findOne({ _id: id }).populate('userId', 'email');

  if (!provider) {
    throw new ApiError(404, 'Provider not found');
  }

  // SECURITY: Validate required verification documents
  const requiredVerifications = ['identity', 'business', 'background'] as const;
  const missingDocs = requiredVerifications.filter((v) => {
    const status = provider.verificationStatus[v];
    return status && status.status !== 'approved';
  });

  if (missingDocs.length > 0) {
    logger.warn('Provider approval blocked - missing verification documents', {
      providerId: id,
      missingVerifications: missingDocs,
      action: 'PROVIDER_APPROVAL_BLOCKED',
    });
    throw new ApiError(400, `Cannot approve: Missing ${missingDocs.join(', ')} verification documents.`);
  }

  // Update provider verification status
  provider.verificationStatus.overall = 'approved';
  if (provider.verificationStatus.adminNotes !== undefined) {
    provider.verificationStatus.adminNotes = notes || 'Provider approved by admin';
  }
  provider.instagramStyleProfile.isVerified = true;
  await provider.save();

  // Update user account status
  if (provider.userId) {
    await User.findByIdAndUpdate(provider.userId, { accountStatus: 'active' });
  }

  // CRITICAL: Create Service documents for each provider service
  if (provider.services && provider.services.length > 0) {
    logger.info('Creating service documents for approved provider', {
      action: 'ADMIN_PROVIDER_APPROVAL',
      providerId: id,
      businessName: provider.businessInfo.businessName,
      serviceCount: provider.services.length,
    });

    for (const service of provider.services) {
      const existingService = await Service.findOne({
        providerId: provider.userId,
        title: service.title,
      });

      if (!existingService) {
        await Service.create({
          providerId: provider.userId,
          providerProfileId: provider._id,
          title: service.title,
          description: service.description,
          category: service.category,
          basePrice: service.basePrice,
          duration: service.duration,
          status: 'active',
        });
      }
    }
  }

  // EMIT SOCKET EVENT: Notify provider in real-time
  const socketServer = getSocketServer();
  if (socketServer) {
    socketServer.emitProviderApproved(
      id,
      new Date(),
      'Your provider account has been approved! You can now start accepting bookings.'
    );
  }

  // CREATE IN-APP NOTIFICATION
  const notificationService = new NotificationService();
  try {
    await notificationService.createNotification({
      recipientId: provider.userId?._id?.toString() || provider.userId?.toString(),
      type: 'provider_approved',
      title: 'Provider Account Approved',
      message: 'Congratulations! Your provider account has been approved.',
      actionText: 'View Dashboard',
      actionUrl: '/provider/dashboard',
      metadata: {
        providerId: id,
        approvedBy: adminUser._id,
        approvedAt: new Date(),
      },
    });
  } catch (notifError) {
    logger.error('Failed to create provider approval notification', {
      providerId: id,
      error: notifError instanceof Error ? notifError.message : String(notifError),
    });
    // Don't fail the main operation if notification fails
  }

  // SEND EMAIL NOTIFICATION
  if (provider.userId?.email) {
    try {
      await sendProviderApproval(provider.userId.email, {
        businessName: provider.businessInfo.businessName,
        approvedAt: new Date(),
      });
    } catch (emailError) {
      logger.error('Failed to send approval email', { providerId: id });
    }
  }

  // AUDIT LOG
  logger.info('ADMIN_AUDIT: Provider approved', {
    action: 'PROVIDER_APPROVED',
    providerId: id,
    providerEmail: provider.userId?.email,
    adminId: adminUser._id,
    adminEmail: adminUser.email,
    timestamp: new Date(),
  });

  res.json({
    success: true,
    message: 'Provider approved successfully',
    data: {
      providerId: id,
      verifiedAt: new Date(),
    },
  });
});
```

```typescript
export const rejectProvider = asyncHandler(async (req: Request, res: Response) => {
  const { id } = req.params;
  const { reason, notes } = req.body;

    'incomplete-documentation',
    'invalid-credentials',
    'business-verification-failed',
    'background-check-failed',
    'non-compliance',
    'other',

    throw new ApiError(400, 'Invalid rejection reason');

      id,
      reason,
      true // canAppeal
```

```typescript
/**
 * Currency formatting utility for the Homeservice platform
 * Uses AED (UAE Dirham) as the default currency
 */

export interface FormatPriceOptions {
  currency?: string;
  showCurrencySymbol?: boolean;
  minimumFractionDigits?: number;
  maximumFractionDigits?: number;
  locale?: string;
}

/**
 * Format a number as currency
 * @param amount - The amount to format
 * @param options - Formatting options
 * @returns Formatted currency string (e.g., "AED 1,234")
 */
export const formatPrice = (
  amount: number,
  options: FormatPriceOptions = {}
): string => {
  const {
    currency = 'AED',
    showCurrencySymbol = true,
    minimumFractionDigits = 0,
    maximumFractionDigits = 0,
    locale = 'en-AE',
  } = options;

  // Handle negative amounts (for refunds, etc.)
  const isNegative = amount < 0;
  const absAmount = Math.abs(amount);

  const formatter = new Intl.NumberFormat(locale, {
    style: showCurrencySymbol ? 'currency' : 'decimal',
    currency,
    minimumFractionDigits,
    maximumFractionDigits,
  });

  const formatted = formatter.format(absAmount);
  return isNegative ? `-${formatted}` : formatted;
};

/**
 * Format amount without currency symbol (just number)
 */
export const formatAmount = (amount: number): string => {
  return formatPrice(amount, { showCurrencySymbol: false });
};

/**
 * Format as compact currency (e.g., "AED 1.2K", "AED 3.5M")
 */
export const formatPriceCompact = (amount: number): string => {
  const absAmount = Math.abs(amount);
  const isNegative = amount < 0;

  let formatted: string;
  if (absAmount >= 1000000) {
    formatted = `${(absAmount / 1000000).toFixed(1)}M`;
  } else if (absAmount >= 1000) {
    formatted = `${(absAmount / 1000).toFixed(1)}K`;
  } else {
    formatted = absAmount.toFixed(0);
  }

  return isNegative ? `-AED ${formatted}` : `AED ${formatted}`;
};

/**
 * Parse a currency string back to number
 */
export const parseCurrency = (value: string): number => {
  // Remove currency symbols and commas
  const cleaned = value.replace(/[^0-9.-]/g, '');
  return parseFloat(cleaned) || 0;
};

/**
 * Validate if a string is a valid currency amount
 */
export const isValidAmount = (value: string): boolean => {
  const amount = parseCurrency(value);
  return !isNaN(amount) && amount >= 0;
};
```

```typescript
// Example: acceptBooking with optimistic update
acceptBooking: async (bookingId: string, data?: BookingAcceptData): Promise<void> => {
  const bookingService = (await import('../services/BookingService')).default;

  // Store previous state for potential rollback
  const previousProviderBookings = [...get().providerBookings];
  const previousCurrentBooking = get().currentBooking;

  // OPTIMISTIC UPDATE: Apply change immediately
  set((state) => {
    // Update the specific booking
    const bookingIndex = state.providerBookings.findIndex(b => b._id === bookingId);
    if (bookingIndex !== -1) {
      state.providerBookings[bookingIndex].status = 'confirmed';
    }

    // Update stats optimistically
    if (state.providerBookingsStats) {
      state.providerBookingsStats.pending--;
      state.providerBookingsStats.confirmed++;
    }

    state.isSubmitting = true;
    state.errors = [];
  });

  try {
    // Call API
    const response = await bookingService.acceptBooking(bookingId, data);
    const updatedBooking = response.data.booking;

    // Reconcile with server response
    set((state) => {
      const bookingIndex = state.providerBookings.findIndex(b => b._id === bookingId);
      if (bookingIndex !== -1) {
        state.providerBookings[bookingIndex] = updatedBooking;
      }
      if (state.currentBooking?._id === bookingId) {
        state.currentBooking = updatedBooking;
      }
      state.isSubmitting = false;
    });
  } catch (error) {
    // ROLLBACK: Restore previous state on failure
    set((state) => {
      state.providerBookings = previousProviderBookings;
      state.currentBooking = previousCurrentBooking;
      state.isSubmitting = false;
      state.errors = [{
        message: error instanceof Error ? error.message : 'Failed to accept booking',
        code: 'ACCEPT_BOOKING_ERROR',
      }];
    });

    // Show error toast
    addToast({
      title: 'Failed to Accept Booking',
      description: error instanceof Error ? error.message : 'Please try again',
      variant: 'error',
    });

    throw error;
  }
}
```
